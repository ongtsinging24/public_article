# 人工智能 理論基礎 — 六層學習路線圖

> 從數學底層到系統工程，由下而上。下層是上層的語言，跳層會在某個點上卡死。

```
第六層：系統與工程   分散式訓練 · 量化/剪枝 · Inference 優化 · MLOps
第五層：訓練典範     預訓練/微調 · RLHF · 遷移學習 · 提示工程 · RAG
第四層：現代架構     Transformer · 注意力機制 · BERT · GPT · Diffusion
第三層：深度學習核心  反向傳播 · CNN · RNN/LSTM · 正規化 · 損失函數
第二層：傳統機器學習  監督/非監督 · SVM · 決策樹 · 貝葉斯 · 集成學習
第一層：數學基礎     線性代數 · 微積分 · 機率統計 · 最佳化理論
```

縮寫對照（abbreviations）：
- **SVD** = Singular Value Decomposition 奇異值分解
- **MLE** = Maximum Likelihood Estimation 最大似然估計
- **KL** = Kullback–Leibler divergence 散度
- **SGD** = Stochastic Gradient Descent 隨機梯度下降
- **CNN / RNN / LSTM** = Convolutional / Recurrent / Long Short-Term Memory Network
- **QKV** = Query / Key / Value（注意力三矩陣）
- **RLHF** = Reinforcement Learning from Human Feedback 人類回饋強化學習
- **RAG** = Retrieval-Augmented Generation 檢索增強生成
- **PEFT / LoRA** = Parameter-Efficient Fine-Tuning / Low-Rank Adaptation
- **MoE** = Mixture of Experts 專家混合
- **KV cache** = Key-Value cache（推論時的注意力快取）
- **MLOps** = Machine Learning Operations

---

## 第一層：數學基礎（最重要，不能跳）

整個領域的底層語言。這層不紮實，上層全是「會調 API 但不知道為什麼」。

- **線性代數（Linear Algebra）**：矩陣乘法、特徵分解（eigen-decomposition）、SVD。
  - 神經網路前向傳播本質就是「矩陣乘法 + 非線性」反覆堆疊。
  - SVD 是 PCA 降維、低秩近似（LoRA 的數學根基）、推薦系統的共同骨幹。
- **微積分（Calculus）**：梯度（gradient）、偏導、鏈式法則（chain rule）、Jacobian / Hessian。
  - 反向傳播 = 鏈式法則的工程化實作，這一條打通其餘自然懂。
- **機率統計（Probability & Statistics）**：貝葉斯定理、MLE、最大後驗（MAP）、KL 散度、熵（entropy）、交叉熵（cross-entropy）。
  - 分類問題的損失函數（cross-entropy loss）直接源自最大似然。
  - KL 散度貫穿 VAE、RLHF（policy 不能離 reference 太遠）、知識蒸餾。
- **最佳化理論（Optimization）**：梯度下降、SGD、動量（momentum）、Adam / AdamW、學習率排程、凸 vs 非凸、鞍點（saddle point）問題。
  - 理解「為什麼大模型用 AdamW 而不是純 SGD」、為什麼要 warmup。

**動手點**：用純 NumPy 手刻一個兩層 MLP 的前向 + 反向，不依賴框架。能跑通，第一三層就通了一半。

---

## 第二層：傳統機器學習（Classical ML）

別跳過。這層的觀念在深度學習時代「完全」適用，而且更容易看清原理。

- **監督學習**：線性/邏輯回歸、SVM（核技巧 kernel trick）、決策樹、KNN。
- **非監督學習**：K-means、階層式聚類、PCA、GMM（高斯混合）。
- **貝葉斯方法**：樸素貝葉斯、貝葉斯推論的思維方式。
- **集成學習（Ensemble）**：Bagging、Random Forest、Boosting（XGBoost / LightGBM）—— 表格資料（tabular）至今的王者，量化/金融特徵建模常用。
- **核心觀念（跨層通用）**：
  - **Bias–Variance tradeoff** 偏差-方差權衡
  - **過擬合 / 欠擬合**、正則化（L1/L2）、交叉驗證（cross-validation）
  - 評估指標：precision / recall / F1 / ROC-AUC，以及它們在類別不平衡下的陷阱。

**動手點**：scikit-learn 跑一個完整 pipeline（特徵工程 → 訓練 → 交叉驗證 → 評估），親手感受過擬合。

## 第三層：深度學習核心（Deep Learning Core）

- **反向傳播（Backpropagation）**：數學推導**真正手算過一次**——這是整層的分水嶺。理解計算圖（computational graph）與自動微分（autodiff）。
- **CNN**：卷積、感受野（receptive field）、平移不變性（translation invariance）、池化。經典：LeNet → AlexNet → ResNet（殘差連接 residual，解決深層梯度消失）。
- **RNN / LSTM / GRU**：序列建模、梯度消失/爆炸、LSTM 的遺忘門（forget gate）設計動機。雖被 Transformer 取代，但其「狀態」思維仍重要。
- **正規化與穩定訓練**：BatchNorm、LayerNorm（Transformer 用這個）、Dropout、殘差連接、梯度裁剪（gradient clipping）。
- **損失函數**：MSE、Cross-Entropy、對比損失（contrastive）、Focal Loss。理解「損失函數 = 你真正在優化的目標」。
- **激活函數**：ReLU / GELU / SiLU，為何深層網路放棄 sigmoid。

**動手點**：PyTorch 從零訓一個 CNN 跑 MNIST/CIFAR，再手刻一次反向傳播驗證和 autograd 結果一致。

## 第四層：現代架構（Modern Architectures）

- **Transformer**：當前所有大模型的基礎。原始論文 **Attention Is All You Need (2017)** 必讀。
  - **注意力機制（Self-Attention）**：QKV 計算要能**從頭推導**——`Attention(Q,K,V) = softmax(QKᵀ/√d_k)·V`。理解 `√d_k` 縮放、multi-head 的意義、位置編碼（positional encoding：絕對 / RoPE 旋轉式 / ALiBi）。
  - 計算複雜度 O(n²)：這是長序列的瓶頸，串起第六層的 FlashAttention 與各種 efficient attention。
- **編碼器 vs 解碼器**：
  - **BERT**（encoder-only）：雙向、masked LM，適合理解/分類/檢索。
  - **GPT**（decoder-only）：自回歸（autoregressive）、causal mask，適合生成。當前主流 LLM 走這條。
  - **T5 / BART**（encoder-decoder）：翻譯、摘要。
- **MoE（Mixture of Experts）**：稀疏激活，用路由（router）只啟動部分參數——現代大模型擴規模的關鍵（如 Mixtral、DeepSeek）。
- **生成模型家族**：
  - **Diffusion Model**：前向加噪 / 反向去噪，圖像生成主流（Stable Diffusion、DALL·E）。
  - **VAE / GAN**：理解生成模型的演進脈絡與各自取捨。

**動手點**：手刻一個 minimal self-attention（參考 Karpathy 的 nanoGPT），把 QKV、causal mask、softmax 全部跑通。

## 第五層：訓練典範（Training Paradigms）

對 LLM 應用開發（如 ROMA 系統）這層最實用——你多半不會自己預訓練，但要懂分界與取捨。

- **預訓練（Pre-training）**：自監督（self-supervised），在海量語料上學 next-token prediction。代價極高，scaling law（Chinchilla 最優算力-資料配比）決定規模策略。
- **微調（Fine-tuning）**：
  - **全參微調 vs PEFT**：LoRA / QLoRA 只訓低秩增量，省顯存——這是第一層 SVD 低秩思想的直接應用。
  - **指令微調（Instruction Tuning / SFT）**：教模型「聽指令」。
- **對齊（Alignment）**：
  - **RLHF**：reward model（從人類偏好對學打分）+ PPO 優化 policy，KL 懲罰拉住不跑偏。
  - **DPO（Direct Preference Optimization）**：跳過顯式 reward model，直接從偏好對優化，更簡單穩定，近年主流。
- **遷移學習（Transfer Learning）**：預訓練表徵遷到下游任務的通用範式。
- **推論期增強（不改權重，最常用）**：
  - **提示工程（Prompt Engineering）**：zero/few-shot、**思維鏈 CoT（Chain-of-Thought）**、ReAct（推理+工具）。
  - **RAG（檢索增強生成）**：embedding → 向量庫檢索 → 注入 context。解決「知識更新 / 幻覺 / 私有資料」三大痛點，是企業落地最高頻方案。理解 chunking、embedding 模型選型、reranker、檢索品質才是 RAG 成敗關鍵。
  - **Agent / Tool Use / Function Calling**：讓模型呼叫外部工具、規劃多步——ROMA 這類系統的核心。

**動手點**：搭一個最小 RAG（文件 → 切塊 → embed → 檢索 → 餵 LLM），再對比「有/無檢索」的回答品質。

## 第六層：系統與工程（Systems & Engineering）

模型能不能上線、跑得起、扛得住流量，全在這層。研究做出來 ≠ 能服務。

- **分散式訓練（Distributed Training）**：
  - **資料平行（Data Parallel / DDP）**：每張卡一份模型、切資料。
  - **模型/張量平行（Tensor Parallel）**、**管線平行（Pipeline Parallel）**：模型放不下單卡時切模型。
  - **ZeRO / FSDP**：切分 optimizer state / 梯度 / 參數，省顯存訓超大模型。
  - 通訊原語：all-reduce、NCCL；瓶頸常在通訊而非算力。
- **模型壓縮（Compression）**：
  - **量化（Quantization）**：FP16 / BF16 / INT8 / INT4（GPTQ、AWQ）。精度換顯存與速度，推論部署核心。
  - **剪枝（Pruning）** 與 **知識蒸餾（Distillation）**：大模型教小模型（用第一層的 KL 散度當對齊目標）。
- **推論優化（Inference Optimization）**：
  - **KV cache**：自回歸生成避免重算歷史 token 的 K/V——理解它才懂為何「長 context 吃顯存」。
  - **FlashAttention**：IO-aware 的注意力實作，省顯存+加速，直接緩解第四層 O(n²) 瓶頸。
  - **PagedAttention / vLLM**、**continuous batching**：提升吞吐（throughput）與 GPU 利用率的服務端關鍵。
  - **投機解碼（Speculative Decoding）**：小模型起草、大模型驗證，降延遲。
  - 指標：latency（TTFT 首 token / TPOT 每 token）vs throughput 的權衡。
- **MLOps / LLMOps**：
  - 資料/模型版本控管、訓練可重現性、CI/CD。
  - **可觀測性（Observability）**：prompt/response logging、token 成本、評估（eval）pipeline、A/B 測試。
  - 線上監控：飄移（drift）、幻覺率、護欄（guardrails）。

**動手點**：用 vLLM 或 Ollama 本地起一個量化模型的推論服務，量測不同量化等級（FP16 vs INT4）的顯存佔用與吞吐差異。

## 經典教材：《深度學習》花書（The "Flower Book"）

**書名**：*Deep Learning*（中譯《深度學習》）
**作者**：Ian Goodfellow、Yoshua Bengio、Aaron Courville（2016, MIT Press）
**俗稱**：中譯本封面是花，故稱「花書」；英文版封面叫「the Deep Learning book」。被視為深度學習領域的**奠基聖經**，理論最嚴謹、數學最完整的一本。線上免費全文：deeplearningbook.org。

### 全書三部分結構

| 部 | 章節 | 內容 | 對應本文層級 |
|--------|----------|---------------------------------|--------------|
| Part I  | Ch 2–5   | 應用數學與 ML 基礎              | 第一、二層   |
| Part II | Ch 6–12  | 現代實用深度網路（核心）       | 第三層       |
| Part III| Ch 13–20 | 深度學習研究（理論/生成模型）  | 進階研究     |

### 分章導讀與閱讀建議

**Part I — 數學與 ML 基礎（對應第一、二層）**
- **Ch 2 線性代數** ✅ 必讀：最精煉的線代速覽，SVD/特徵分解一次到位。數學生疏者當複習。
- **Ch 3 機率與資訊理論** ✅ 必讀：機率、熵、KL 散度——後面損失函數與生成模型全靠它。
- **Ch 4 數值計算** ⭐ 重要：條件數、梯度爆炸/消失的數值根源、上溢/下溢，工程踩坑的理論解釋。
- **Ch 5 機器學習基礎** ✅ 必讀：bias-variance、容量、正則化、MLE、最大後驗——**全書最該精讀的一章**，把第二層觀念講透。

**Part II — 現代實用深度網路（對應第三層，落地最相關）**
- **Ch 6 深度前饋網路** ✅ 必讀：MLP、反向傳播完整推導、激活函數。手算反向傳播看這章。
- **Ch 7 正則化** ✅ 必讀：L1/L2、Dropout、early stopping、資料增強——調模型的核心武器庫。
- **Ch 8 最佳化** ✅ 必讀：SGD、動量、Adam、batchnorm、初始化策略，理論與實務兼顧。
- **Ch 9 CNN** ⭐ 重要：卷積、池化、感受野，做視覺/結構化資料必讀。
- **Ch 10 RNN / LSTM** ⭐ 重要：序列建模、梯度消失、LSTM 門控設計動機。
- **Ch 11 實務方法論** ✅ 高 CP 值：如何選超參、debug、決定下一步——少見地談「工程直覺」，常被低估。
- **Ch 12 應用** 🔵 選讀：視覺/語音/NLP 案例，2016 視角，當歷史脈絡看。

**Part III — 深度學習研究（進階，可按需深挖）**
- **Ch 13–15 線性因子模型 / 自編碼器 / 表徵學習** ⭐ 想懂 embedding、表徵學習、VAE 前身讀這幾章。
- **Ch 16–19 結構化機率模型 / 蒙地卡羅 / 配分函數 / 近似推論** 🔴 數學最重，做機率生成模型研究才需要，應用導向可跳。
- **Ch 20 深度生成模型** ⭐ VAE、GAN、Boltzmann machine——生成模型的理論源頭（Diffusion 在本書後出現，需另補）。

### 重要提醒：花書的時間侷限
> 花書定稿於 **2016 年，早於 Transformer（2017）**。書中**沒有** Transformer、注意力機制、BERT/GPT、Diffusion、RLHF、RAG 這些內容（對應本文第四～六層）。
>
> 因此花書的定位是：**打穩第一～三層的理論地基**。第四層以後請接原始論文（Attention Is All You Need 等）與較新教材，例如 Bishop《Deep Learning: Foundations and Concepts》(2024)、d2l.ai《動手學深度學習》（含 Transformer、附程式碼）、Karpathy 的 nanoGPT/影片系列。

### 怎麼用這本書（依目標分流）
- **想紮實打底/走研究**：Ch 5 → Ch 6 → Ch 7/8 精讀，Part I 當數學手冊隨查。
- **LLM 應用開發者（如 ROMA）**：Ch 5、6、7、8、11 讀懂即可，Part III 跳過，省下的時間投到第四～六層的新材料。
- **完全別「從第一頁讀到最後一頁」**：花書是參考書（reference），不是小說。按本文六層的缺口針對性補。

## 學習順序與心法

1. **不能跳第一層**。數學不通，上面全靠背。
2. **每層都要手動跑通一個最小實作**，光看會的是錯覺，手算過/跑通過才是真懂。
3. **由下而上理解、由上而下實作**：先建立全圖，再按你的目標（NLP / RL / 生成 / LLM 應用）深挖某一支。
4. **對 LLM 應用開發者**：第一層打底 → 第四層懂 Transformer 原理 → 重押第五（RAG/Agent/微調）與第六層（部署/成本/優化）。中間層理解即可、不必精通。
